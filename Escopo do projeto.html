Viso Geral do Projeto

O LeadFlow uma plataforma para gesto completa de leads, eventos e vendas. Conecta gestores de eventos, clientes (empresas), vendedores e recepo num ecossistema integrado com Facebook Ads API, WhatsApp Business e automaes via webhooks para n8n.

Integraes Externas

Facebook Ads API
  • Cliente conecta conta Facebook via OAuth 2.0
  • Backend armazena access_token + page_id + ad_account_id
  • Lead Ads Webhook: lead preenche formulrio → POST automtico ao backend
  • Backend processa: cria lead, atribui campanha, dispara webhook n8n
  • Sincronizao peridica de campanhas via Marketing API
Webhooks para n8n

HTTP POST disparado automaticamente para a URL configurada pelo cliente:

EventoDescrio
lead.createdNovo lead cadastrado
lead.stage_changedLead mudou de etapa no CRM
lead.assignedLead atribudo a vendedor
lead.confirmedLead confirmou presena
lead.checked_inCheck-in na recepo
event.createdNovo evento criado
campaign.startedCampanha iniciada
conversation.new_messageNova mensagem no chat
vendor.course_completedVendedor concluiu curso
facebook.new_leadLead do Facebook Ads
// Exemplo de payload webhook { "event": "lead.created", "timestamp": "2026-03-17T14:30:00Z", "client_id": "cli_abc123", "data": { "lead_id": "lead_xyz789", "name": "Joo Silva", "phone": "+5511999999999", "source": "facebook_ads", "campaign_id": "camp_def456" } }
WhatsApp Business API
  • Integrao via WhatsApp Cloud API (Meta) — oficial
  • Receber e enviar mensagens, templates aprovados, mdia
  • Nmero configurado na aba Conexes do cliente

Riscos e Mitigaes

NvelRiscoMitigao
ALTO Atraso nas credenciais Facebook/WhatsApp Solicitar com antecedncia (Sprint 3). Usar mocks at chegarem.
ALTO Escopo maior que 60h disponveis Priorizar features crticas. Secundrias viram backlog ps-lanamento.
MDIO Mudanas na API do Facebook Usar verso estvel da API. Monitorar changelog.
MDIO Performance com volume alto de leads Paginao, cache Redis, ndices PostgreSQL desde Sprint 3.
BAIXO Rejeio nas stores (Apple/Google) Seguir guidelines desde o incio. Buffer para ajustes.
BAIXO Indisponibilidade do n8n Retry com backoff exponencial. Logs para reenvio manual.

Critrios de Aceite

  • Todas as tarefas do sprint implementadas e demonstradas

  • APIs funcionais e testveis via Postman/Insomnia

  • Sem erros crticos (crash) nas funcionalidades entregues
  • Demonstrao de 15–30 minutos ao final do sprint
  • Bloqueios comunicados imediatamente (no no final do sprint)
  • Documentao bsica (README, endpoints) atualizada
